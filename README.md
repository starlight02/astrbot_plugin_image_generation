# 通用生图插件 (Image Generation)

![:name](https://count.getloli.com/@astrbot_plugin_image_generation?name=astrbot_plugin_image_generation&theme=miku&padding=7&offset=0&align=top&scale=1&pixelated=1&darkmode=auto)

面向 AstrBot 的通用图像生成插件，支持多供应商、文生图、图生图、LLM 工具调用、预设、限流和安全审核。

> [!warning]
> 使用v1.1.0+版本的用户,请暂时不要使用人设添加功能！！！  
> 请等待[#8228](https://github.com/AstrBotDevs/AstrBot/pull/8228)修复合并到主线后再使用  
> 否则附带file类型的嵌套配置会导致webui卡死！！！  
> 升级至v1.1.0+时请先备份配置文件，由于配置结构变动，已添加的预设提示词会在升级过程中丢失！！！

## 功能

- 多供应商配置：一个插件中配置多个生图服务，每个供应商可独立设置 API Key、Base URL、代理和模型列表。
- 模型动态切换：通过 `/生图模型` 查看和切换模型，模型格式为 `供应商名称/模型名称`。
- 文生图与图生图：自动识别消息图片、引用消息图片和 @ 用户头像作为参考图。
- LLM 工具调用：可作为 LLM 工具在对话中自动触发生图。
- 预设与人设：支持提示词预设，也支持带固定描述和参考图的人设模板。
- 稳定性控制：支持多 API Key 轮询、重试、并发限制、频率限制和每日额度。
- 安全审核：支持提示词屏蔽词、AI 提示词审核、AI 图片审核和审核白名单。

## 命令

| 命令                                              | 说明                                                       |
| :------------------------------------------------ | :--------------------------------------------------------- |
| `/生图 <预设/人设...> [额外提示词] [数量]` 或 `/生图 <提示词> [数量]` | 生成图片。开头连续命中的预设/人设会按空格分隔并依次应用；末尾单独数字表示生成数量；消息、引用消息、@ 用户或人设参考图会作为参考图。 |
| `/生图任务`                                       | 查看当前会话正在进行的生图任务，并显示编号。               |
| `/生图任务 <编号或任务ID>`                        | 查看指定进行中生图任务详情。                               |
| `/生图取消 <编号或任务ID>`                        | 取消当前会话中仍在排队或运行的生图任务。                   |
| `/生图模型`                                       | 查看可用模型和当前模型。                                   |
| `/生图模型 <序号>`                                | 切换到指定序号的模型。                                     |
| `/预设`                                           | 查看所有预设和人设。                                       |
| `/预设 添加 <预设名:预设内容>`                    | 添加预设。                                                 |
| `/预设 删除 <预设名>`                             | 删除预设。                                                 |

## 配置

### 基础配置

| 配置名        | 默认值                               | 说明                                                                                                                                                          |
| :------------ | :----------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 启用 LLM 工具 | 生图工具、预设查询工具、预设编辑工具 | 选择允许 LLM 调用的工具。生图工具同时支持普通生图、预设生图、自拍/人设生图和图生图；预设查询工具用于预设/人设列表与详情查询；预设编辑工具用于创建和删除预设。 |

### API 供应商配置

每个供应商都包含以下通用配置,部分供应商会有特殊配置项

| 配置名           | 说明                                                          |
| :--------------- | :------------------------------------------------------------ |
| 适配器名称       | 供应商显示名，也是模型格式 `供应商名称/模型名称` 的前半部分。 |
| API Base URL     | 供应商接口地址。多数适配器可留默认值或填写中转地址。          |
| 代理地址         | HTTP 代理，例如 `http://127.0.0.1:7890`。                     |
| API Keys         | API Key 列表；多个 Key 会自动轮询和重试。                     |
| 可用模型列表     | `/生图模型` 中展示和切换的模型列表。                          |
| 模型能力         | 按实际模型能力勾选文生图、图生图、宽高比、分辨率。            |
| 生图超时时间覆盖 | 单个供应商的超时时间；填 0 使用公共生图配置。                 |
| 最大重试次数覆盖 | 单个供应商的重试次数；填 0 使用公共生图配置。                 |


### 生图配置

| 配置名               | 默认值               | 说明                                                                                            |
| :------------------- | :------------------- | :---------------------------------------------------------------------------------------------- |
| 生图模型             | 自动选择首个可用模型 | 当前使用的模型，可通过 `/生图模型` 更新。                                                       |
| 生图超时时间         | 180 秒               | 请求超时时间。                                                                                  |
| 最大重试次数         | 3                    | 生成失败后的最大重试次数。                                                                      |
| 最大并发生图请求数   | 3                    | 同时调用图像模型接口的请求数量上限；多图任务会在该限制内并发请求，完成一个后继续补发剩余请求。  |
| 默认出图数量         | 1                    | 未在指令末尾或 LLM 工具中指定数量时，单个生图任务默认生成的图片数量。                           |
| 单次最大出图数量     | 10                   | 单个生图任务最多生成的图片数量；用户请求超过该值时会自动截断。                                  |
| 默认图片宽高比       | 不指定               | 可选不指定、1:1、2:3、3:2、3:4、4:3、4:5、5:4、9:16、16:9、21:9；不指定时请求中不携带比例字段。 |
| 默认分辨率           | 不指定               | 可选不指定、1K、2K、4K；不指定时请求中不携带分辨率字段，仅部分适配器支持。                      |
| 显示结果信息         | 用量                 | 发送图片后可附加耗时、模型、生成数量、用量和任务ID；用量仅在开启每日额度时显示。                |
| 开始生图任务提示模板 | 已开始生图任务       | 生图开始时发送的提示文本；留空则不发送，默认包含任务ID。                                        |

开始生图任务提示模板支持这些占位符：`{reference_image_count}`、`{image_count}`、`{count}`、`{prompt}`、`{preset}`、`{presets}`、`{persona}`、`{personas}`、`{aspect_ratio}`、`{resolution}`、`{task_id}`、`{model}`、`{mode}`、`{reference_images_block}`、`{image_count_block}`、`{count_block}`、`{preset_block}`、`{persona_block}`。其中 `{preset}`/`{presets}` 仅表示命中的预设，`{persona}`/`{personas}` 仅表示命中的人设。

### 任务体系

`/生图` 指令和 LLM 生图工具共用统一任务体系：任务提交后会生成 `task_id`，进入排队/运行/完成/失败/取消状态；模型接口调用受最大并发生图请求数限制。多图任务会在请求并发限制内同时发起多张图片生成，已有请求完成后继续补发剩余请求，所有请求完成后统一发送结果。`/生图任务` 只展示当前会话正在进行的任务，并为列表项添加编号；可通过 `/生图任务 <编号或任务ID>` 查看详情，通过 `/生图取消 <编号或任务ID>` 取消仍在排队或运行中的任务。任务记录保存在内存中，插件重载后会清空；已结束任务不会在列表中展示。

### 用户限制配置

| 配置名             | 默认值                 | 说明                                                                                         |
| :----------------- | :--------------------- | :------------------------------------------------------------------------------------------- |
| 会话黑名单         | 空                     | 黑名单会话无法使用指令和 LLM 工具。                                                          |
| 管理员无视使用限制 | 开启                   | AstrBot 管理员跳过会话黑名单、频率限制和每日额度，但仍记录每日用量；成功提示中显示为 `n/∞`。 |
| 使用限制白名单 UMO | 空                     | 额外配置可跳过使用限制的会话 UMO，行为同管理员绕过。                                         |
| 黑名单拦截提示     | 当前会话未启用生图功能 | 黑名单命中时返回的文本，留空则不提示。                                                       |
| 请求频率限制       | 0 秒                   | 单个会话两次请求之间的最小间隔；0 表示不限。                                                 |
| 参考图大小限制     | 10 MB                  | 超过限制的参考图会被忽略。                                                                   |
| 每日生图数量限制   | 关闭                   | 是否限制每日生成数量。                                                                       |
| 每日允许生成数量   | 10                     | 开启每日限制后，每个会话每天可生成的最大数量。                                               |

### 安全审核配置

| 配置名                     | 说明                                                             |
| :------------------------- | :--------------------------------------------------------------- |
| 审核白名单                 | 白名单会话跳过提示词和图片审核。                                 |
| 自定义屏蔽词               | 提示词命中任意词会被直接拒绝。                                   |
| 启用 AI 提示词审核         | 生图前调用模型审核提示词。                                       |
| 提示词审核模型 Provider ID | 指定审核模型；留空使用当前会话模型。                             |
| 提示词审核提示词           | 审核提示模板，支持 `{prompt}` 占位符。                           |
| 启用 AI 图片审核           | 生图后调用模型审核生成图片。                                     |
| 图片审核模型 Provider ID   | 指定审核模型；留空使用当前会话模型。                             |
| 图片审核提示词             | 审核图片的提示模板；只有显式写入 `{prompt}` 时才附带生图提示词。 |

### 提示词模板配置

预设提示词和人设模板在同一个配置组中维护。

| 配置名       | 说明                                         |
| :----------- | :------------------------------------------- |
| 预设提示词   | 预定义提示词模板，可通过 `/预设` 命令增删。  |
| 人设名称     | 用于 `/生图 <人设名称...> [额外提示词]` 匹配。 |
| 人设文本描述 | 使用该人设生图时会作为基础提示词。           |
| 人设参考图   | 可选；当前模型支持图生图时会自动作为参考图。 |

简单预设格式：`名称:提示词`。

高级预设格式：`名称:{"prompt":"提示词","aspect_ratio":"16:9","resolution":"2K","description":"描述"}`。

`/预设` 会同时展示预设和人设。预设与人设同名时，`/生图` 优先使用预设。`/生图` 会从提示词开头按空格连续解析多个预设或人设，遇到第一个未命中的词后，剩余内容作为额外提示词；末尾单独数字会被解析为生成数量，不参与预设/人设和提示词内容。LLM 生图工具可通过 `preset` 参数使用一个或多个预设，也可通过 `persona` 参数使用一个或多个人设，多个名称可用空格分隔；并支持 `prompt`、`aspect_ratio`、`resolution`、`avatar_references` 和 `reference_images` 参数。LLM 生图工具只使用显式传入的参考图和人设参考图，不会自动读取当前消息图片、引用图片或 @ 头像，并会在提交任务前去除重复参考图。LLM 预设查询工具默认只返回预设或人设名称列表，也可以通过名称查看具体内容。LLM 预设编辑工具可创建和删除预设，创建时支持简单提示词格式和高级 JSON 格式。

## 使用示例

基础生图：

```text
/生图 一只在森林里野餐的兔子
```

使用预设：

```text
/预设 添加 赛博朋克:cyberpunk style, neon lights, futuristic city
/生图 赛博朋克 猫
```

使用人设：

```text
/生图 看板娘 在咖啡店读书
```

组合多个预设/人设：

```text
/生图 手办化 看板娘 夜景 微笑
```

一次生成多张：

```text
/生图 手办化 看板娘 夜景 微笑 3
```

图生图：

```text
发送或引用一张图片后输入：/生图 变成动漫风格
/生图 变成像素风 @用户A
```

切换模型：

```text
/生图模型
/生图模型 2
```

查看或取消任务：

```text
/生图任务
/生图任务 1
/生图任务 1a2b3c4d
/生图取消 1
/生图取消 1a2b3c4d
```

## 注意事项

- 火山方舟的可用模型列表需要按控制台实际 Model ID 或 Endpoint ID 配置。
- 配置 `jimeng2api` 后，插件会在启动时和每天凌晨自动领取积分；仅直接连接即梦逆向时有效。
- 即梦逆向图生图使用 `/v1/images/compositions`，使用中转可能会导致图生图失败。
- 欢迎提交 Issue 或 PR 添加新的适配器。
