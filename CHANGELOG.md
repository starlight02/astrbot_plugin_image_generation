### 更新日志
- **v1.3.2-2026-06-07**
  - 修复自定义 HTTP 适配器无需 API Key 时仍被指令、LLM 工具和公共 API 拦截的问题。
  - 优化生图任务交付状态与发送失败处理，任务记录会区分生成完成、发送完成和分批发送失败。
  - 新增每日额度预占与结算机制，降低并发任务超出每日额度的风险。
  - 生成结果会按真实图片类型保存为对应扩展名，并可配置清理取消、失败或审核不通过任务的临时图片。
  - 同步配置重载后的运行时组件设置，并统一参考图 `ImageData` 输入归一化逻辑。
  - 提取模板解析、任务 ID 和结果格式化工具，简化主插件职责并移除未使用的 `PREPARING` 任务状态。
  - 新增公共 API 返回码枚举并补充返回码文档，便于其他插件稳定判断调用结果。

- **v1.3.1-2026-06-07**
  - 加强参考图安全校验：本地参考图仅允许读取当前会话 workspace、AstrBot temp 目录和本插件数据目录内的文件。
  - 新增参考图真实文件类型校验，拒绝非图片内容，避免将任意本地文件作为参考图发送给远程生图服务。

- **v1.3.0-2026-06-06**
  - 新增 `custom_http` 自定义 HTTP JSON 生图适配器，支持自定义 URL、请求头、查询参数、请求体和响应图片提取路径。
  - 自定义 HTTP 和 OpenAI Chat 的 JSON 配置项启用代码编辑器模式，便于维护复杂 payload。
  - 将原 `gemini_openai` 适配器重命名并改造为通用 `openai_chat` OpenAI Chat Completions 兼容适配器，不再保留旧配置键兼容。
  - OpenAI Chat 适配器支持自定义提示词前缀、`modalities` 和额外请求体 JSON。
  - 新增 `/生图` 空参数帮助信息：展示当前使用模型和常用指令用法，替代原本的提示词缺失提示。
  - 优化 `/生图任务` 列表末尾用法提示，拆分查看详情和取消任务说明，提高可读性。

- **v1.2.9-2026-06-05**
  - 合并 [PR #29](https://github.com/Railgun19457/astrbot_plugin_image_generation/pull/29),添加Agnes AI适配器
  - 移除配置中关于人设模板的警告
  - 调整最低AstrBot版本至`4.25.3`

- **v1.2.8-2026-05-30**
  - 新增插件间公共 API：其他插件可通过 `plugin.public_api` 提交生图任务、查询状态、取消任务、等待结果并获取生成图片本地路径。
  - 新增公共 API 使用文档，说明接口参数、返回对象和调用示例。
  - 优化生图任务查询：`/生图任务 <任务ID>` 和 LLM 任务管理工具可查看已结束但尚未清理的任务详情，任务列表仍只展示正在进行的任务。

- **v1.2.7-2026-05-29**
  - 修复指令生图和 LLM 工具生图开始提示、任务创建日志中的参考图数量显示不准确问题。
  - 调整参考图处理顺序：提交任务前先完成参考图收集与去重，再发送开始通知并创建任务，确保提示和任务记录使用真实参考图数量。
  - 简化任务创建流程，移除后台参考图准备回调，降低参考图计数与任务状态维护复杂度。

- **v1.2.6-2026-05-29**
  - 优化生图任务失败前置检查：生成器未初始化、API Key 缺失、提示词审核失败等场景不再误消耗冷却时间。
  - 优化参考图处理流程：参考图下载、头像获取移动到后台任务内，新增“准备参考图”任务状态，并统一指令入口和 LLM 工具入口的参考图收集与去重逻辑。
  - 优化多图生成结果处理：复用任务级参考图转换结果，按 `max_images_per_message` 配置分批发送结果，并保留适配器单次返回的全部图片。
  - 新增全局不可重试错误配置，仅保留不可重试 HTTP 状态码和错误关键词；命中后立即失败，其余错误按重试次数继续尝试。
  - 改进任务结果统计与详情：生成后图片审核失败也计入实际生成额度，LLM 工具任务取消时不再唤醒 AI，任务详情展示每个子请求的状态和重试次数。

- **v1.2.5-2026-05-28**
  - 多图任务改为在最大并发生图请求数限制内并发请求，完成一个请求后继续补发剩余请求，所有请求完成后统一发送结果。
  - 移除插件内置配置迁移逻辑，改为仅统一校验配置内容是否合理；选择类和数值类配置异常时自动恢复默认值。
  - 新增审核模型重试次数配置，提示词审核和图片审核在模型调用失败、返回为空或无法解析结果时会按配置重试，默认 3 次。

- **v1.2.4-2026-05-28**
  - 新增单任务多图生成：指令和 LLM 生图工具均支持指定生成数量，任务内按单张图片独立重试并汇总返回结果。
  - 新增 `/生图` 指令数量写法：末尾单独数字表示生成数量，和开头连续预设/人设解析保持一致。
  - 将最大并发限制调整为请求级别，限制同时调用图像模型接口的请求数量，而不再限制整个生图任务数量。
  - 新增默认出图数量和单次最大出图数量配置；每日额度按实际成功生成图片张数记录，并在提交前按请求张数校验剩余额度。

- **v1.2.3-2026-05-28**
  - 新增 LLM 生图任务工具 `manage_image_tasks`，支持 AI 查看当前会话的生图任务列表、任务详情并取消正在进行的任务。
  - 生图任务详情新增当前重试次数展示，便于排查任务执行进度和重试状态。
  - 将生图任务工具接入 LLM 工具开关配置，并兼容旧版默认配置自动启用。

- **v1.2.2-2026-05-28**
  - 优化 LLM 生图工具任务完成后的结果交付流程：任务完成后会将结果返回给 AI，由 AI 使用 `send_message_to_user` 工具发送给用户。
  - 支持在模型具备图像输入能力时将生成图片一并提供给 AI 查看；不支持图像输入时仅提供生成图片路径。
  - 新增 AI 结果处理失败时的直接发送兜底，避免生成成功但用户收不到结果。
  - 重构 LLM 工具结果处理逻辑到独立模块，保持 `main.py` 简洁并降低维护成本。
  - 优化 LLM 工具提交任务成功时的返回文本，明确区分“任务已提交”和“图片已生成”。
  - 清理冗余参数和死代码，并修复 AI 已成功发送结果后仍触发模型空输出重试日志的问题。

- **v1.2.1-2026-05-27**

  - LLM 生图工具支持在 `preset` 和 `persona` 参数中传入多个名称，同时移除人设未指定比例时默认使用 `9:16` 的行为。

  - 提交优化任务创建、成功日志位置，并移除部分冗余 `debug` 日志，降低运行日志噪音。

  - 新增 `/生图任务`、`/生图任务 <编号或任务ID>` 和 `/生图取消 <编号或任务ID>` 指令，用于查询和取消正在进行的任务。

  - 降低成功路径日志噪音：适配器调用、参数规范化、图片转换、参考图去重和结果发送等过程细节改为 `debug`。
